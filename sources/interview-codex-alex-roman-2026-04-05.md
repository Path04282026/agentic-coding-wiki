# Interview — Codex — Alex & Roman (2026-04-05)

**Format:** Podcast interview (with Peter Steinberger as host)
**Speakers:** [Alex (Codex PM)](references/entities/alex-(codex-pm).md) (PM Lead), Roman (DevX Lead)
**Existing summary:** `raw/summaries/CodeX_20260405_Summary.md`

## Key Takeaways by Dimension

### [01 - Agent Loop](references/dimensions/01-agent-loop.md)
- Product should be "almost invisible, get out of the way of the model"
- Three-layer architecture: open-source Rust harness → configurable primitives → simple surface
- Delegation model with progressive intelligence, not fixed agentic loop

### [05 - Multi-Agent Architecture](references/dimensions/05-multi-agent-architecture.md)
- Sub-agents adopted bottom-up by community before official release
- Vision: "infinitely many models doing work independently — validating, deploying, monitoring"
- Peter Steinberger's 18-terminal setup inspired the Codex App

### [06 - UI and Distribution](references/dimensions/06-ui-and-distribution.md)
- Three pivots: Codex Cloud → CLI + IDE → Codex App
- App decision was "quite contentious" — should they even build an app?
- Progressive disclosure design: "like playing a game, discovering what's next"
- Codex on all platforms including Windows

### [07 - Feature Gating and Internals](references/dimensions/07-feature-gating-and-internals.md)
- Community forks features before official release
- Very few specs: "10 bullets or something and that's it"
- Designers write more code than engineers did 6 months ago

### [09 - Philosophical Divergence](references/dimensions/09-philosophical-divergence.md)
- "Near-term or long-term, never medium-term" planning
- PM as "fill-in-the-gaps" not leadership
- "All of the lines between career ladders are blurring — we're all builders"
- Hiring: "Show me what you built" >> credentials

## Notable Quotes

> "How do we mostly let the product be almost invisible, get out of the way of the model."

> "We started seeing crazy things on social media… Peter Steinberger with 18 terminals across three monitors."

> "It was quite contentious — should we even be building an app?"

> "You either plan near-term or long-term, but you never plan medium-term."

> "The designers on the Codex team write more code now than was written by an engineer six months ago."
